# MERCURY Playbook (Revenue-First)

## Decision order

1. Diagnose the single bottleneck hurting paid conversions.
2. Build no more than 5 hypotheses per cycle.
3. Rank by expected conversion lift x confidence minus effort/risk cost.
4. Block any tactic that violates trust/compliance constraints.
5. Package only approval-ready experiments for BRAINIAC.

## Experiment archetypes

- Hook tests: first 2 seconds and angle specificity.
- Problem-language tests: user wording from comments/search.
- Landing tests: above-the-fold proposition clarity.
- Offer tests: risk reversal and checkout friction.
- Proof tests: screen recording + protocol execution evidence.

## Required weekly output

- Weekly bottleneck memo.
- Ranked experiment slate.
- Content and landing test pack.
- Expected conversion impact.
- Evidence citations and confidence labels.
- Explicit approvals required from BRAINIAC.

## Calibration rule

If confidence and observed outcomes diverge for two consecutive cycles:

- lower confidence scores by one band for similar tactics,
- add failure examples to training candidate set,
- prioritize lower-risk experiments until calibration improves.
